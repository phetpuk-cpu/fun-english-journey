# ชุดทดสอบอัตโนมัติ (Unit Test)

## วิธีรัน

```bash
node --test tools/test/*.test.js
```

ไม่ต้อง `npm install` — ใช้ test runner ที่ติดมากับ Node อยู่แล้ว (ต้องใช้ Node 18 ขึ้นไป)

รันเฉพาะไฟล์เดียว:
```bash
node --test tools/test/assets.test.js
```

## เทสต์นี้ตรวจอะไรบ้าง

| ไฟล์ | ตรวจอะไร |
|---|---|
| `engine-pure.test.js` | ฟังก์ชันพื้นฐาน — `slug()` (แปลงคำเป็นชื่อไฟล์), `similarity()` (ให้คะแนนพูด/เขียน), ธงสัญชาติ, ข้อความแจ้งปัญหาไมค์ |
| `engine-pipeline.test.js` | ลำดับขั้นของบทเรียนแต่ละชนิด (ปกติ/นิทาน/อนุบาล), การไปบทถัดไป, แบบทดสอบจำลอง |
| `engine-scoring.test.js` | สูตรคิด XP เต็มและเกณฑ์ดาวของทุกชนิดบท |
| `dashboard.test.js` | ตรรกะแดชบอร์ดผู้ปกครอง — จำนวนบทต่อชั้น, จุดที่ควรฝึกเพิ่ม, สรุปกิจกรรม |
| `assets.test.js` | **ไฟล์ภาพ/เสียงจริงตรงกับเนื้อหาบทเรียนไหม** — จับภาพหาย/ชื่อผิด/ไฟล์เกิน/เสียงซ้ำ |
| `content.test.js` | schema ของไฟล์เนื้อหา, `UNIT_FILES` ตรงกับไฟล์จริง, รหัสบทไม่ซ้ำ, แท็บชั้นครบ |

## สำหรับ Codex / Antigravity — อ่านตรงนี้

**หลัง push ไฟล์ภาพหรือเสียงทุกครั้ง ให้รันเทสต์ก่อน:**

```bash
node --test tools/test/assets.test.js
```

ถ้าแดง แปลว่าไฟล์ที่ส่งมามีปัญหาอย่างใดอย่างหนึ่ง:
- **ชื่อไฟล์ไม่ตรงกับคำในบทเรียน** — เช่นตั้ง `don-t-like` แต่ระบบหา `dont-like`
  (กฎคือ: ตัดอะพอสทรอฟีทิ้ง ไม่แปลงเป็นขีด · ช่องว่างเป็นขีด · ตัวเล็กทั้งหมด)
- **ภาพขาด** — มีคำในบทเรียนแต่ไม่มีไฟล์
- **ไฟล์เกิน** — มีไฟล์ที่ไม่ตรงคำไหนเลย (มักเกิดจากใส่ชื่อบทผิด)
- **ไม่มี `.png` คู่กับ `.webp`** — ต้องมีทั้งคู่เสมอ
- **ไฟล์เสียงซ้ำกัน** — เช่นไฟล์ชื่อตัวอักษรเป็นไฟล์เดียวกับเสียง phoneme

**บน GitHub Actions เทสต์จะรันอัตโนมัติทุกครั้งที่ push และถ้าไม่ผ่าน เว็บจะไม่ถูก deploy**
(กันไฟล์เสียทับของดีบน production)

## ภาพที่ยังผลิตไม่เสร็จ

ถ้ามีภาพที่รู้อยู่แล้วว่ายังไม่เสร็จ ให้ใส่ชื่อไว้ใน `PENDING_IMAGES` ที่หัวไฟล์ `assets.test.js`
เพื่อไม่ให้เทสต์แดง — **และลบออกทันทีเมื่อส่งไฟล์แล้ว** เพื่อให้เทสต์คุมต่อ

## หลักการเขียนเทสต์เพิ่ม

- **ห้ามแก้ไฟล์ใน `fun-english-journey/js/` เพื่อให้เทสต์ผ่าน** — เทสต์โหลดไฟล์ production
  ผ่าน `helpers/load-engine.js` โดยไม่แตะโค้ดจริง (เพราะมีหลายเอเจนต์แก้ไฟล์เดียวกัน)
- **อย่าเขียน logic เลียนแบบ engine ในเทสต์** — ให้เรียกฟังก์ชันจริงเสมอ
  มิฉะนั้นเทสต์จะทดสอบโค้ดของตัวเอง ไม่ได้ทดสอบ production
- **ทุกครั้งที่แก้บั๊ก ให้เพิ่มเทสต์ที่จับบั๊กนั้นได้** พร้อมคอมเมนต์ว่าเคยพังยังไง
