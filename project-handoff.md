### 3.3 Deploy ขึ้นเว็บจริง (ฟรี) — ตรวจสอบความพร้อมแล้ว พบ 2 จุดต้องแก้ก่อน deploy จริง
- ไมโครโฟน (SpeechRecognition/MediaRecorder) ต้องรันบน **https** เท่านั้น — GitHub Pages เป็น https อยู่แล้ว ✅ ไม่มีปัญหา
- **⚠️ ปัญหาที่ 1 — symlinks:** `fun-english-journey/assets/img` และ `/audio` เป็น git symlink (ตรวจแล้วด้วย `git ls-files -s` เจอ mode `120000`) ชี้ไปที่ `English_fun/assets/img|audio` — **GitHub Pages ไม่ resolve symlink ให้** (serve เป็น broken link ไม่ใช่ไฟล์จริง) ถ้า deploy ตรงๆ ภาพ/เสียงทั้งหมดจะพังบนเว็บจริงแม้จะรันได้ปกติบน local
- **⚠️ ปัญหาที่ 2 — ตำแหน่งไฟล์:** ตัวแอปจริงอยู่ใน `English_fun/fun-english-journey/` (subfolder) ไม่ใช่ root ของ repo (`English_fun/`) — GitHub Pages โหมด "Deploy from a branch" serve ได้แค่จาก root หรือ `/docs` เท่านั้น ไม่รองรับ subfolder อื่นตามใจชอบ
- **แนวทางแก้ที่แนะนำ (ยังไม่ได้ทำ รอคุยกับผู้ใช้ก่อน):** ใช้ GitHub Actions workflow สร้าง build step ที่ copy ไฟล์จริงแทน symlink แล้ว publish ไปยัง branch `gh-pages` แยกต่างหาก (ไม่ต้องย้ายโครงสร้างโปรเจกต์เดิม, symlink ยังใช้งานได้ปกติสำหรับ Codex/Antigravity เขียนไฟล์เข้ามาต่อ)
- **สิ่งที่ต้องขออนุญาตก่อนทำจริง:** การสร้าง GitHub repo ใหม่ (`gh repo create`) และ `git push` ขึ้น remote — จะถามผู้ใช้ก่อนทำทุกครั้ง ไม่ทำเองแบบเงียบๆ

### 3.4 เสียบไฟล์ภาพ/เสียงจริงแทน placeholder — ทำงานอัตโนมัติอยู่แล้ว ✅

### 3.6 กิจกรรมแปลงประโยค (Sentence Transform) — เพิ่งเพิ่ม ยังนำร่องแค่ 3 บท
เพิ่ม step type ใหม่ `"transform"` ใน engine.js (เห็นได้จาก field `transform` ในไฟล์ JSON ที่เป็น array แบบ optional ต่อบทเรียน) — ทดสอบแล้วว่า backward compatible 100% (บทเรียนที่ไม่มี field นี้ทำงานเหมือนเดิมทุกอย่าง ไม่พัง) ตอนนี้มีเนื้อหาจริงแค่ 3 บท: `p3u7l2` (Past Simple statement→negative/question), `p5u4l1`, `p5u4l2` (must/mustn't) — ถ้าผลตอบรับจากลูกชายดี ค่อยขยายไปบทอื่นที่มีไวยากรณ์เหมาะสม (เช่น present perfect ป.6, passive voice ป.6)
